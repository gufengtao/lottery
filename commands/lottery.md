# Lottery Command - 双色球生成器

生成符合特定规则的双色球号码，自动避免与历史 300 期重复。

## 参数说明

| 参数              | 说明           | 默认值      |
| --------------- | ------------ | -------- |
| `--last-red`    | 上期红球号码（逗号分隔） | 自动从历史数据获取 |
| `--last-blue`   | 上期蓝球号码       | 自动从历史数据获取 |
| `--repeat`      | 重复上期红球数量     | 0-2（随机） |
| `--consecutive` | 连续号码组数       | 0-2 组（随机，1-2 组常见） |
| `--same-tail`   | 同尾号组数        | 0-1 组（随机，约 50% 概率） |

## 规则说明

1. **红球**：6 个，范围 1-33
2. **蓝球**：1 个，范围 1-16
3. **默认规则**（改进版）：
   - 红球：
     - 0-2 个红球重复上期（随机）
     - 0-2 组连续号码（约 60-70% 概率出现）
     - 0-1 组同尾号（约 50% 概率出现）
     - **前 20 期热度规则**：优先选择前 20 期出现过的号码（80% 概率）
   - 蓝球：
     - 尽量不跟上期重复
     - 考虑大小/奇偶交替规律（约 60% 概率反向选择）
4. **历史 300 期不能重复**

## 统计规律参考

> 以下规律基于历史数据统计特征，仅供生成号码时参考，不保证中奖。

### 红球统计特征

| 特征 | 常见范围/比例 | 说明 |
|------|-------------|------|
| **奇偶比** | 3:3 或 4:2 / 2:4 | 极端比例 (5:1、6:0、1:5) 后下期大概率回归均衡 |
| **大小比** (小 1-16, 大 17-33) | 3:3 或 2:4 / 4:2 | 连续偏向一侧后容易回调 |
| **三区比** (一区 01-11、二区 12-22、三区 23-33) | 2:2:2 ~ 3:2:1 ~ 1:3:2 | 连续某区超多 (如 4:1:1) 后容易修正 |
| **和值** (6 红相加) | 80-130 (平均 105-110) | 极端值 (<70 或>140) 后容易回归中值 |
| **连号** | 1-2 组 (概率≈60-70%) | 3 连以上较少 |
| **重号** (与上期重复) | 出现概率≈40-50% | 有时连续几期有，有时突然断档 |
| **冷热号** | 热号短期延续，冷号回补 | 冷号可能遗漏 15-30 期后回补 |

### 蓝球统计特征

| 特征 | 常见范围/比例 | 说明 |
|------|-------------|------|
| **奇偶** | 奇数≈55% | 波动较大，大偶→小奇常见回调 |
| **大小** (小 1-8, 大 9-16) | 大号略多 | 近期常出现大偶→小奇回调 |
| **交替规律** | 约 60% 概率反向 | 大→小、小→大、奇→偶、偶→奇交替 |
| **质合** (质数 1,2,3,5,7,11,13) | 合数稍多 | |
| **重复/同尾** | 偶尔连续重复或同尾跳动 | 如 4→14→4 |
| **冷热回补** | 遗漏 10 期 + 回补概率上升 | 不是必然 |

### 应用建议

- 生成号码时可适当考虑**奇偶均衡**、**三区分布**
- 避免极端和值，选择 80-130 范围
- **连号**：1-2 组较为合理（约 60-70% 概率出现）
- **同尾号**：非必然出现，约 50% 概率
- **蓝球**：可参考奇偶、大小交替规律

### 前 20 期热度规则（重要）

> **规则**：本期开出的红球号码，通常至少在前 20 期内出现过。

| 特征 | 说明 |
|------|------|
| **热度范围** | 前 20 期 |
| **规则描述** | 本期 6 个红球应全部在前 20 期出现过 |
| **例外情况** | 极少数情况下可能出现 1-2 个"冷门"号码（遗漏 20 期+） |
| **应用场景** | 生成号码时，优先从前 20 期出现过的号码中选择 |

**原理**：彩票号码存在"热度延续"现象，完全陌生的冷号（遗漏 20 期以上）同时开出多个的概率极低。

## 数据来源

- **数据来源**: 中国福彩网官方 (kaijiang.zhcw.com)
- **历史数据路径**: `../data/lottery-history.json` (插件目录内)
- **存储内容**: 最近 300 期开奖号码（包含期号、开奖日期、红球、蓝球）
- **自动清理**: 超出 300 期的记录自动删除
- **数据更新**: 使用 `/lottery-update` 命令从官网同步最新数据

## 相关命令

```bash
/lottery         # 生成新一期号码
/lottery-update  # 从官网更新历史开奖数据
```

### 格式规范

| 字段 | 格式 | 说明 |
|------|------|------|
| **期号** | 6 位数字，右对齐 | 如 `2026029` |
| **日期** | MM-DD 格式 | 如 `03-17` |
| **红球** | 6 个号码，两位数字 + 空格分隔，居中 17 字符 | 如 `06 19 22 23 28 31` |
| **蓝球** | 2 位数字，右对齐 | 如 `05` |
| **位置图** | 16 个位置，每个号码占 4 字符宽度 | 从 01-16 依次排列，蓝球放在对应位置 |

### 位置图生成规则

```python
# 蓝球位置图生成逻辑
positions = []

    if i == blue_ball:
        positions.append(f'{blue_ball:02d}')  # 显示蓝球号码
    else:
        positions.append('  ')  # 空白
# 每个位置占 4 字符，用空格连接
position_map = ' '.join(positions)
```
